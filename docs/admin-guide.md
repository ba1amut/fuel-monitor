# Fuel Monitor — Руководство администратора

## Мерж дублирующихся станций

### Когда мержить

На карте (`fuel.weatherpath.ru`) видны два маркера в одном месте или в радиусе 50–200 м, относящиеся к одной физической АЗС. Это происходит, когда разные пользователи указали немного разные названия станции и автоматическое сопоставление не сработало.

### Как найти ID станций

Открыть карту и кликнуть на маркер — в попапе под названием виден короткий ID (первые 8 символов UUID). Полный UUID отображается в tooltip при наведении. Также можно получить список всех станций:

```bash
curl -s https://fuel.weatherpath.ru/api/stations | python3 -m json.tool | grep -E '"id"|"aliases"'
```

### Порядок мержа

1. Определить **source** (дубль, который нужно удалить) и **target** (основная запись, которая останется).
2. Вызвать эндпоинт:

```bash
curl -X POST https://fuel.weatherpath.ru/api/stations/{SOURCE_ID}/merge-into/{TARGET_ID} \
  -H "X-Admin-Key: ${ADMIN_KEY}"
```

3. Проверить результат — эндпоинт возвращает обновлённую target-станцию с объединёнными алиасами и топливными состояниями.

### Что происходит при мерже

- Все **репорты** source переназначаются на target.
- **Алиасы** source добавляются к алиасам target (дубли убираются).
- **Топливные состояния** source мержатся в target: если grade уже есть в target — оставляется более свежая запись; если нет — переносится из source.
- **Координаты**: если у target нет GPS, берутся от source.
- **Source-станция удаляется** из БД.

### Переменная окружения

`ADMIN_KEY` задаётся в `.env` на сервере. Без неё или с неверным ключом эндпоинт возвращает 403.
